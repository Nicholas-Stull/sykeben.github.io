Introduction

So, you wanna learn Markdown?

Markdown (or md files) is a document language mostly used by developers for its simplicity and amazing ability to be cross-platform. Markdown is used by developers around the world and is a great language to learn as all of GitHub supports it.

In this tutorial, you will learn the basics of Markdown, why it's used, and how to use it.

What's StackEdit?

To make editing Markdown easier, we'll need an editor.

For this, we'll use StackEdit, a free, web-based markdown editor. StackEdit saves your edited files to your browser, so they'll remain saved there until you reset it, or clear out the history.

If you worried about them using your data, read their privacy policy. They promise your information will be safe.

Preparing the Workspace

Before you get started, let's clean up.

Since you probobly haven't visited StackEdit yet, it's going to be cluttered. Let's clean it up. First, open the folder menu (A1), right click on the welcome file (A2), and click delete (A3). Now, click new file (B), type in a name like readme.md (C) and press Enter⏎.

With the file created, let's clear it out. Click at the end of the editor (D), and hold backspace until all the text is gone. Once it's cleared, the file is ready.

Basic Formatting

Let's write some Markdown!

Before writing, let's quickly review StackEdit. There are 3 "sections", the File Menu, Editor, and Preview Window. These are pretty self-explanitory.

There are 3 basic formats, normal, italic, and bold. Normal text has no formatting brackets around it, it's normal like this (A). Italic text is surrounded by 1 astrik, like *this* (B). And, bold text is surrounded by 2 astriks, like **this** (C).
