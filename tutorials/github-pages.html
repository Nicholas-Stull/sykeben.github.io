Getting Started with GitHub Pages

 1/?
Introduction

GitHub pages is a great way to show off your skills as a developer, publish documentation, cheaply host fully custom websites, and more.

In fact, the website you are currently reading is a GitHub pages site (more on this in the next slide). The whole reason this site is different than, let's say, a Google Site, is the fact it fully supports custom HTML, CSS, JS, and whatever else you may want in your site. It's like having a free webserver, but you can link it to you, and your repositories!

Example

A great example of using GitHub pages to make a real, working, site is what your looking at now!

In fact, this whole webpage is being hosted on GitHub pages, even with custom HTML, CSS, JS, and proper Bootstrap! The thing to left, the carousel, is a custom component provided by Bootstrap. The code that makes this text change with the carousel when you click Next → is a custom JS script.

The reason this site works and looks the way it does is proof that GitHub pages is a great alternative to running a standalone web server.

Preparing a Repo

Before we can start writing your Pages, we need to prepare a new repository.

First, on your GitHub dashboard, click your user icon (A1), and then click "Your repositories" in the dropdown (A2). This will bring you to your repository list, there click "New" (B). Now, you're creating a repository! Give it a name (C1), a description (C2), check the "README" box (C3), and click "Create Repository" (C4).

If you arrive at a page similar to the one shown (D), you've succesfully created a repository and are ready to move onto the next step.

Writing the README

So, now you have a repo, it's time to write something!

To begin editing the README.md file, which is the "homepage", we need to open the editor. Click the pencil next to the README preview (shown in the image) to begin.

Click inside the editor box, and remove all the text to make sure we have a clean slate. Now, type this exact code:
# My First Page
**Hello, World!** This is my first page!

This is the new contents of the document. It is written in Markdown, which you can learn about here.

Saving Your Changes

Now that we've written something, we have to save it.

Saving things in Git is called "committing" them. This means you "commit" to the changes you've made and you want to "push" them to the repository. By clicking the "Commit changes" button at the bottom (shown in the image), you "commit" the file, and save it. Click the "commit" button now.

Enabling the Site

With the README ready, it's now time to enable the GitHub Pages site.

To get to the place where you can enable it, head to the "Settings" tab (1). Now that you're there, under "GitHub Pages", click the source selection dropdown (2), and select "master branch" (3). This tells GitHub Pages to use the whole repository as a webpage, turning that README.md file we wrote earlier into the homepage.

If this is all done correctly you should see a message in a blue bar that says "GitHub Pages source saved." at the top of your screen.

Look, Ma!

Now that we've "published" your site, let's go check it out.

To view your newly created page type a URL formatted like the one below into your addressbar and hit go.

For example, if your username was therock, and the name of the repo is epic-site, then the URL would be: therock.github.io/epic-site.
